在 `joint_recommendation_q3.py` 代码中，得出推荐设定值的过程并非简单的“四舍五入”或“阈值截断”，而是一个基于**代价函数最小化（Cost Function Minimization）**的**离散最优投影**过程。

以下是该离散映射逻辑的详细步骤：

### 1. 定义离散可行域 (Candidate Space)
代码首先预设了控制变量的所有可能离散取值。由于 $DFlow$ 和 $DGap$ 的调节档位通常为 -1、0、1，代码通过 `np.meshgrid` 生成了一个包含 9 种可能组合的“决策网格”：

* **可行集**：$\mathcal{S} = \{(-1, -1), (-1, 0), (-1, 1), (0, -1), (0, 0), (0, 1), (1, -1), (1, 0), (1, 1)\}$
* **代码体现**：
    ```python
    allowed_f = np.array([-1, 0, 1])
    allowed_g = np.array([-1, 0, 1])
    grid = np.array(np.meshgrid(allowed_f, allowed_g)).T.reshape(-1, 2)
    ```

### 2. 构建连续空间的“锚点”
在进行映射之前，模型先计算出两个“理想中的连续值”作为参考：
* **数据驱动预测值**：由训练好的 XGBoost 模型给出的预测值 $\hat{F}$ 和 $\hat{G}$（这是纯粹基于数据拟合得到的连续浮点数）。
* **协同逻辑值**：基于第二问的主从回归方程，计算出在当前脉冲特征下，$DGap$ 理论上应该对应 $DFlow$ 的哪个协同位置。

### 3. 计算离散组合的代价 (Weighted Loss)
这是映射的核心。对于每一个样本，代码会遍历上述 9 种离散组合，并计算每种组合的“损失得分”。



`weighted_loss` 函数（代码第 75 行）计算了以下三个维度的偏差：
1.  **$DFlow$ 偏差**：候选离散值 $F$ 离 XGBoost 预测值 $\hat{F}$ 有多远。
2.  **$DGap$ 偏差**：候选离散值 $G$ 离 XGBoost 预测值 $\hat{G}$ 有多远。
3.  **协同约束偏差**：候选组合 $(F, G)$ 是否符合第二问定义的物理联动方程（$G = \beta F + c(x)$）。

**数学表达式：**
$$Loss(F, G) = \lambda_1(F - \hat{F})^2 + \lambda_2(G - \hat{G})^2 + \lambda_3(G - (\beta F + c(x)))^2$$

### 4. 最优决策投影 (Minimum Loss Selection)
代码通过循环计算这 9 个 Loss 值，并使用 `np.argmin` 选出 Loss 最小的那一个组合。

* **投影逻辑**：哪个离散坐标点能让这三个偏差的加权总和最小，模型就“投影”到哪个点。
* **代码体现**（第 135-144 行）：
    ```python
    # 对每个离散候选组合计算 loss
    losses = [
        weighted_loss(cand_f, cand_g, f_hat, g_hat, cx, prior_beta, lambdas)
        for cand_f, cand_g in grid
    ]
    # 选取损失最小的索引
    best_idx = np.argmin(losses)
    # 最终输出推荐值
    rec_f, rec_g = grid[best_idx]
    ```

### 为什么这样做比“阈值映射”更好？
1.  **多目标权衡**：如果单纯对 $DFlow$ 做阈值判定，可能会得到一个虽然离散化很准，但严重违反 $DGap$ 联动逻辑的值。现在的做法是在“准”和“稳（协同）”之间找最优平衡。
2.  **抗噪性**：如果预测值在 0.5 附近摆动，传统的阈值映射会非常敏感。通过引入 $\lambda_3$（协同项），模型会倾向于选择那些符合物理逻辑的档位，从而过滤掉部分随机噪声。
3.  **结果可执行性**：这种方法保证了输出结果严格落在 $\{-1, 0, 1\}$ 内，直接生成可供车间操作员使用的决策建议。

**总结**：代码通过在离散的九宫格内搜索使“数据偏差”与“物理约束偏差”之和最小的点，完成了从连续预测到离散设定的最优映射。
